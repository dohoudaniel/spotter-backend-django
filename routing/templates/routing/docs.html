How it works

Stations are geocoded once at import from an offline dataset, so the request path makes exactly one external call (to OSRM for the route). Corridor matching and the fuel optimization are pure in-memory computation. The fuel choice is the classic gas-station problem: buy just enough to reach the next cheaper station in range, otherwise fill up and run to the cheapest reachable one — total gallons is fixed by distance, so only where you buy is optimized.

Try it: curl "http://127.0.0.1:8000/route/?start=Dallas,%20TX&finish=Chicago,%20IL"
